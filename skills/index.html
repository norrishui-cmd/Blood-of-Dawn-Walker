What Skill Pages Should Answer

Players need to know which early upgrades are safe, which skills define a build, and whether mistakes can be reset. Those answers require hands-on testing, so this page defines the data structure first.

Skill Data Fields

  • Skill name, category, unlock condition, and upgrade cost.
  • Combat, exploration, stealth, or dialogue role.
  • Scaling, cooldown, resource cost, and synergy.
  • Recommended builds and beginner priority.
  • Respec availability and refund cost.

Internal Link Plan

Every skill should link to at least one build, one combat use case, and any quest or boss where it matters. That keeps the site useful instead of becoming a static list.
