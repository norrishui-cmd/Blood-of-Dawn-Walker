Build Philosophy

Before ability scaling and gear stats are known, the best guide approach is to define playstyle templates rather than pretend to know optimal damage numbers.

Starter Templates

Survival Build
Prioritizes healing, defense, safer exploration, and forgiving combat choices.
Burst Damage Build
Focuses on ending fights quickly, but may require precise timing and resource control.
Stealth Route Build
Values scouting, ambushes, escape tools, and quest flexibility.
Narrative Choice Build
Tracks dialogue, faction outcomes, and non-combat utility if those systems exist.

What Needs Testing

  • Respec availability and cost.
  • Ability scaling by weapon, stat, or story progression.
  • Best early upgrades for new players.
  • Boss-specific build counters.

Launch-Week Publishing Pipeline

When a build is tested, copy the build template from templates/build-template.html. Publish only after skill costs, power synergies, and respec risk are known. Link each build to combat, powers, and tested boss pages.

FAQ

Can best builds be published before launch?
Not as final rankings. Pre-launch build pages should be frameworks that explain playstyles and the evidence needed before recommending exact setups.
What build types are useful to prepare?
Prepare beginner, survival, stealth, vampire, melee, burst, control, and story-focused build pages so launch data has a clear destination.
What data makes a build guide trustworthy?
Skills, costs, gear, respec rules, boss matchups, route risk, and tested strengths and weaknesses make build advice trustworthy.
